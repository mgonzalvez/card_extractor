# Martin's Card Extractor - Progress Log
Date: 2026-02-11

## Direction Update

The project is now intentionally **grid-first** (manual, reliable slicing) instead of auto-detect-first.

Primary UX goal: make extraction predictable and easy for general PnP users by letting them define one accurate grid profile and reuse it across the document.

## Current Product State

### Implemented
- Site name and branding updated to **Martin's Card Extractor**.
- High-legibility visual styling and cleaned page header copy.
- Static, client-side architecture (GitHub Pages compatible, no backend).
- PDF upload and per-page canvas preview/editing.
- Page role assignment: `front`, `back`, `mixed`.
- Grid slicer workflow implemented:
  - inputs for rows, columns, margins, gutters, nudges
  - **Initialize Grid on This Page**
  - on-canvas draggable grid lines for refinement
  - **Apply Grid to All Pages** using one profile for both fronts and backs
- Export pipeline:
  - PNG output
  - ZIP packaging into `fronts/` and `backs/`
  - optional single-back export (one back file)
  - Build ZIP progress indicator and download-ready visual state
- Favicon added.

### Kept but De-emphasized
- Auto-detect and manual box tools remain available under **Advanced / Experimental**.
- Auto-detect messaging updated to point users toward grid standardization.

## Known Issues / Active Gaps

1. Front alignment quality can still vary on some layouts if initial grid setup is off.
2. Fine-grained handle/line interaction should continue to be tuned for less misclick frustration.
3. Legacy template/calibration code paths still exist in `app.js` though the primary UI is now grid-first.

## Agreed Constraints (Current)

- One grid profile for now (most PnP PDFs share the same grid on each page).
- Duplex assumption for v1: same grid profile works for front and back pages.
- Nudge adjustments included.
- Keep optional single-back checkbox.
- Hide auto-detect in Advanced/Experimental.

## Next Priorities

### P1
- Improve usability of grid line drag targets (desktop + touch) to fully resolve misclick/dismiss behavior.
- Tighten grid alignment defaults and guardrails so first-pass setup is more accurate.

### P2
- Remove or refactor unused template/calibration logic to reduce complexity and avoid UX confusion.
- Add concise in-app helper text/tooltips for each grid input (margin/gutter/nudge semantics).

### P3
- Expand acceptance checks against `samples/` and `sample output/` to verify alignment consistency before release.
